New in 3.8.
Added minimum and maximum bet limits to slots.table for the slot machine, matching the same system as blackjack. The min_bet key rejects any bet below the threshold. The max_bet key silently clamps bets above the limit. Setting max_bet to 0 allows unlimited bets.
Updated ranks.table to include a blackjack unlock entry at rank 10, moving the cooky flipper unlock to rank 20 and the slot machine unlock to rank 30. The ranks.table comment header now documents all three unlock values.
Fixed a bug where the amount bet was not being subtracted from the player's total before the round began.
Made it so the card scores and ace counts reset correctly at the end of every round, whether you win, lose, or push.
Made it so you can no longer bet more than the amount of the item you currently hold.
Made it so you are required to place a bet before cards are dealt.
Made it so you don't have to stand every time you get 21 points. The game now automatically resolves the round when your score hits 21.
Updated jacks.table to reload alongside all other configuration files when Control L is pressed, both in the main game and inside each minigame.
Added minimum and maximum bet limits to jacks.table. The min_bet key rejects any bet below the threshold. The max_bet key silently clamps bets above the limit. Setting max_bet to 0 allows unlimited bets. Both apply regardless of which item is being bet.
Added support for betting any item you want in blackjack. This matches the same system as the slot machine. You can bet cookies, coins, auto cookies, auto cookie speeds, or manual cookies. Wins, losses, naturals, and pushes all apply to whichever item you chose. Baking speed bets correctly adjust the bake interval in both directions.
New in 3.7.
Added a fully configurable jacks.table file for blackjack. This supports setting the dealer stand threshold, payout multipliers for naturals, wins, and pushes, minimum and maximum bet limits, sounds for each event, and messages for every outcome.
Made it so the dealer stands at 17 by default, matching standard casino rules.
Made it so a natural blackjack, 21 on the opening two cards, pays out at 1.5x the bet. Standard wins pay 2x. Ties return the original bet.
Added hit and stand actions to blackjack. Hitting draws another card. Standing triggers the dealer to draw until reaching the stand threshold, then the scores are compared.
Moved the cooky flipper unlock to rank 20 and the slot machine unlock to rank 30 to make room for blackjack at rank 10.
Added blackjack as a new minigame, unlocked at rank 10. Players bet an item of their choice and receive two cards to start.
The cooky store menu and the upgrades sub-menu now remember your last selected position when you return to them, completing full position memory across the entire shop system. All four levels of the shop, the store menu, the upgrades menu, the singles category menu, the singles item menu, the bundles category menu, and the bundles item menu, now restore focus to where you left off.
Added a new optional %actual% placeholder to the event description format in baker.event and flipper.event. For percentage based events, %actual% is replaced with the real computed stat change rather than the percentage value. This makes event messages more informative by showing both the percentage and the exact amount gained or lost. If omitted from a description the message remains concise as before. All ten built-in percentage events in baker.event have been updated to include the %actual% placeholder. Both baker.event and flipper.event now document both the %amount% and %actual% placeholders in their comment headers.
Updated the flipper event descriptions to report exact values rather than vague wording. Events that previously said things like you gained some auto-cookies now say you gained %amount% auto-cookies so the player always knows the precise stat change. The baking speed events also now report the exact millisecond value gained or lost.
Updated the dlg sounds with new ones.

Fixed a bug in the bundle shop parser where the description field was bleeding into the quantity of the last item in the bundle, causing values like 2:A well-rounded bundle to appear in the menu label. The shop shows each bundle's dynamic cost calculated from the current shop prices of its component items, so the price scales naturally with your progression just like the normal shop. Bundles can be locked behind a minimum rank. When purchasing a bundle you are asked how many you want to buy, and the total cost is the per-bundle price multiplied by that quantity. You can type a lower number to buy less of each item, or leave it at the default to spend as much as possible. Menus without a rank or description continue to work as before.

Items are assigned to a category by placing the alias as the first field on their line. Adding a new category requires only a new alias definition and no script changes. Pressing back or escape from a category returns you to the category menu rather than all the way out to the store.
New in 3.4.
Updated the todo list file with the latest bug fix and feature improvements made to the game.
Fixed a long standing bug where the auto slot production and manual slot production cost multiplier sliders in the game settings menu displayed a value of 0.099999999999989X instead of 1X on opening. This was caused by the default multiplier value of 1.001 losing precision when converted to a slider position. The defaults are now 1.01, which maps cleanly to 1X with no floating point errors.
Fixed a bug where stats such as coins, cookies, auto cookies, manual cookies, and baking speed could display fractional values like 0.76 coins due to floating point drift from percentage events or slot machine payouts. All five stats are now floored to whole numbers every game tick in the main game, and immediately after every slot machine win or loss inside the minigame.
Fixed a bug where losing a bet in the slot machine could cause a stat to go negative, resulting in values like 0.76 coins. Stats are now clamped to zero immediately after a loss is applied.
Fixed the slot machine bet confirmation messages. They previously said you placed a bet of X items on coins. They now correctly say you placed a bet of X coins, or the appropriate item name for each bet type.
Fixed a bug where placing a bet in the slot machine at very large quantities could incorrectly fail with a not enough of that item error. The bet is now clamped to your available amount instead of being rejected, matching the same fix applied to cookie selling.
Added a toggle to evenly distribute cookie production across manual slots in the game settings menu. When enabled, your manual cookie output is split across your slots rather than multiplied by them, mirroring the existing option for automated slots.
Fixed a bug where selling cookies at very large quantities could incorrectly fail with a not enough cookies error. This happened because typing the full number back into the input box caused floating point precision loss at large values, making the parsed amount appear slightly higher than the actual cookie count. The sell logic now clamps the typed amount to your available cookies instead of rejecting it, so typing more than you have simply sells everything.
Removed the levcoins variable entirely. Rank rewards are now applied directly to the target stat by the reward system rather than being accumulated in a middleman variable.
Replaced the hardcoded rank reward system with a fully data-driven one defined in data/config/ranks.table. Sounds, default per-rank rewards, and milestone rewards are each configured in their own section, supporting stat targets, rank-scaled amounts, optional unlocks, and message placeholders. Milestone rewards fire alongside the default, with multiple entries per rank chosen at random.

Made the slot machine reel count configurable via the reels= line in slots.table. The default is 5. If you change this value, make sure no payout match value in the same file exceeds the reel count, otherwise that payout can never be triggered.
Replaced the hardcoded slot machine symbol list with a data-driven one. Symbols are now defined as a single comma separated line in data/config/slots.table. The alphabetical symbol ordering setting in game settings continues to work as before, with the unordered mode now shuffling the loaded symbols rather than using a second hardcoded list.
Replaced the hardcoded slot machine payout tiers with a data-driven payout table in slots.table. Each payout line defines the number of matching symbols required, the multiplier applied to the bet, the sound to play, and the message spoken to the player.
The chance field in flipper.event is now respected when an event fires. Setting an event's chance to less than 100 means it may not fire even when selected, and setting it to 0 effectively disables it without removing it from the file or the checkbox list.
Made it so all of the flipper events now share the same underlying logic as the main baker event system, including percentage support, rank operators, the zero stat guard, plurality fixes, multiple sounds with random range syntax, and the alert sound for positive and negative events.
The cooky flipper event list in the game UI is now built dynamically from the loaded flipper events, so any events you add to flipper.event will automatically appear as options in the checkbox list.

Fixed a bug where shop prices were scaling based on the player's current stat values rather than how many times they had actually purchased upgrades. This caused events and minigame winnings that boosted stats for free to make shop items appear far more expensive than intended. Shop pricing now tracks purchases independently for each stat category, so only actual shop purchases affect the cost of future upgrades.
